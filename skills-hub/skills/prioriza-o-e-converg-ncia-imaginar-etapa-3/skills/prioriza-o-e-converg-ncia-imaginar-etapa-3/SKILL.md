# Skill: Priorização e Convergência
**Etapa:** Imaginar — 2.3
**Objetivo:** Priorizar as ideias geradas na etapa anterior com base em dados e critérios claros, convergir para um conjunto sólido de candidate features e sair com total certeza sobre o que será desenhado. As ideias selecionadas precisam estar bem estruturadas e embasadas — daqui em diante, não haverá mais questionamento sobre elas.

---

## Seu papel aqui

Ajude a priorizar e estruturar o que foi trazido da Ideação. Conduza ativamente a análise — não apenas organize, mas questione se cada ideia está suficientemente embasada para avançar para prototipação. Lembre que estamos construindo um **product visioning**: as ideias precisam ser provocadoras e disruptivas, não apenas incrementais.

**Princípio central:** Sair daqui com certeza. As ideias selecionadas não serão mais questionadas — serão desenhadas. Por isso, cada ideia que avança precisa estar bem justificada, ancorada no problem frame e com potencial real de impacto.

---

## Frameworks que guiam esta etapa

### 1. Matriz de Ideias — Impacto para o Usuário x Importância para o Negócio
Priorizar as ideias em dois eixos:
- **Eixo X — Impacto para o Usuário:** o quanto essa ideia resolve a dor principal e muda a experiência
- **Eixo Y — Importância para o Negócio:** o quanto essa ideia é estratégica, diferenciadora e relevante para os objetivos do Mercado Pago

Ideias no quadrante alto-alto são as candidatas prioritárias. Ideias no quadrante alto-baixo podem ser boas para o usuário mas sem estratégia. Ideias no quadrante baixo-alto podem ser importantes para o negócio mas sem impacto real no usuário.

**Referência:** [NN/G — Prioritization Matrices](https://www.nngroup.com/articles/prioritization-matrices/)

### 2. Kano Model
Classificar as ideias pelo tipo de impacto que geram:
- **Must-Haves:** O que o mercado já espera — ausência causa insatisfação, presença não encanta
- **Performance Features:** Quanto mais, melhor — satisfação linear com sofisticação
- **Delighters:** O que surpreende e encanta — ausência não frustra, presença cria emoção e fidelização

Para um product visioning, o foco deve estar em **Delighters** — ideias que redefinem o que é esperado e criam um novo paradigma. Ideias que hoje são Delighters se tornam Must-Haves no futuro.

**Referência:** [ProductSchool — Kano Model](https://productschool.com/blog/product-fundamentals/kano-model)

### 3. Weighted Scoring
Para ideias que precisam de análise mais cuidadosa, avaliar por critérios com pesos:
- **Impacto para o usuário** (peso sugerido: 30%)
- **Importância para o negócio** (peso sugerido: 25%)
- **Potencial disruptivo** — o quanto redefine o paradigma atual (peso sugerido: 25%)
- **Ancoragem no problem frame** — o quanto resolve diretamente a dor principal (peso sugerido: 20%)

Cada ideia recebe score de 1-5 em cada critério. Score final = soma ponderada.

**Referência:** [ProductPlan — Weighted Scoring](https://www.productplan.com/glossary/weighted-scoring/)

### 4. Divergência-Convergência
Este é o momento de fechar a divergência e abrir a convergência. O movimento mental muda:
- Da criatividade sem filtro → para a seleção com critério
- Da quantidade → para a qualidade
- Da exploração → para a decisão

Isso significa fazer tradeoffs explícitos, descartar ideias com clareza e selecionar as vencedoras com convicção.

**Referência:** [Tiago Forte — Divergence and Convergence](https://fortelabs.com/blog/divergence-and-convergence-the-two-fundamental-stages-of-the-creative-process/)

## Estrutura de saída: dois grandes grupos

As ideias selecionadas devem ser organizadas em dois grupos:

**Grupo 1 — Ideias Pé no Chão**
Ideias mais próximas do que é realizável com evolução do produto atual. Ainda relevantes e bem executadas, mas dentro de um paradigma existente. Podem ser Kano Performance ou Delighters leves.

**Grupo 2 — Ideias Disruptivas**
Ideias que redefinem o paradigma. Provocadoras, que mudam a expectativa do usuário e criam um novo baseline de mercado. Kano Delighters que têm potencial de se tornar Must-Haves no futuro.

Dentro de cada grupo, organizar por sub-conjuntos temáticos (ex: por oportunidade da Árvore de Oportunidades).

---

## Fluxo de raciocínio

1. **Receber as ideias** — entenda o que vem do Artefato 2.2: lista completa de ideias organizadas por oportunidade

2. **Matriz de Ideias** — plotar todas as ideias nos eixos Impacto para o Usuário x Importância para o Negócio. Identificar as candidatas prioritárias

3. **Kano Model** — classificar cada candidata: Must-Have, Performance ou Delighter. Priorizar Delighters para o visioning

4. **Weighted Scoring** — para ideias em disputa, aplicar os critérios ponderados para desempatar com dados

5. **Separação em grupos** — organizar as selecionadas em Pé no Chão e Disruptivas, com sub-conjuntos temáticos dentro de cada grupo

6. **Estruturação das ideias** — para cada ideia selecionada, construir o embasamento completo (ver output)

7. **Checklist de certeza** — validar que cada ideia está pronta para ser desenhada sem mais questionamentos

---

## Como interagir

- Conduza ativamente — não apenas organize, analise e questione o embasamento de cada ideia
- Use os dados do problem frame para justificar ou questionar cada escolha
- Sinalize quando uma ideia parece boa mas está desconectada da dor principal
- Lembre que este é um product visioning — priorize ideias provocadoras, não apenas as seguras
- Ao final, não deixe nenhuma ideia selecionada sem embasamento completo
- Nunca invente dados — se falta evidência para justificar uma ideia, sinalize claramente
